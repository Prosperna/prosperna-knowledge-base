### UC 05 | Detect Low Stock after Order Checkout

| Field                           | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID                     | UC 05                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Prepared By                     | Adrianne Berida                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Last Updated                    | October 14, 2025                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Objectives                      | The System should detect when a product's stock quantity becomes equal to or below the global threshold immediately after checkout deduction, and classify it as a Low Stock trigger event. The system should simultaneously trigger an in-app notification (per-product, near real-time) and queue the event for consolidated email batching.                                                                                                                                                                                |
| Actor                           | System                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Preconditions                   | Merchant notifications are enabled. A valid threshold value is configured. Customer successfully checks out a product, and system deducts stock quantity. Merchant is on a Pro or Premium plan.                                                                                                                                                                                                                                                                                                                               |
| Conditions (Optional)           | Applies to all confirmed orders regardless of payment method (Credit Card, COD, etc.). Only applies when change originates from customer orders (not merchant edits). In-app notification sends immediately upon detection; email is queued for 1-minute batching.                                                                                                                                                                                                                                                            |
| Steps                           | Customer places an order that reduces product stock. System deducts the stock quantity of the ordered product variant and location. System evaluates the resulting stock quantity after deduction. If quantity ≤ threshold and > 0, system classifies event as Low Stock. System checks the cooldown record for that variant-location. If cooldown is active, system skips notification creation. If cooldown is inactive, system flags event for batch processing (UC 06) and triggers in-app notification delivery (UC 22). |
| Postconditions                  | A Low Stock trigger event is created and sent to the queue for 1-minute email batching. An in-app notification is sent near real-time, immediately upon detection.                                                                                                                                                                                                                                                                                                                                                            |
| Business Trigger                | Product stock quantity falls to or below threshold immediately after checkout deduction.                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Acceptance Criteria             | Event fires only for non-merchant stock changes. Trigger condition: quantity ≤ threshold and > 0. Event not created if variant-location is under cooldown. In-app notification sent near real-time. Email queued for batch processing.                                                                                                                                                                                                                                                                                        |
| Estimates                       |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Success Message                 |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Error Messages                  |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Business Rules/Desired Behavior | Trigger occurs post-order deduction step. Event metadata includes variant_id, location_id, and merchant_id. Multiple triggers may occur simultaneously per checkout event. In-app notification displays immediately; email batching occurs within 1-minute window. In-app and email notifications are independent streams.                                                                                                                                                                                                    |

| Objectives                      | The System should queue all Low Stock events detected within a fixed 1-minute window and group them per merchant for consolidated email notification. | Steps                           | System receives a Low Stock event. In-app notification has already been sent (UC 22). In-app notifications have been delivered separately.                                                                                                                                                                                                                                                                                                                                                                                           |

| Acceptance Criteria             | Events are successfully grouped within the same 1-minute window. Each merchant receives only one Low Stock batch per window. New triggers after the window starts a new 1-minute timer. In-app notifications do not delay email batching.                                                                                                                                                                                                                                                                                                         |
| Estimates                       |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Success Message                 |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Error Messages                  |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Business Rules/Desired Behavior | Each merchant has an isolated batching queue (e.g., Redis). 1-minute timer is fixed (not sliding). Threshold changes mid-window do not affect ongoing batch (see UC 18). Email batching and in-app notification are separate, independent processes.                                                                                                                                                                                                                                                                                              |
