# Introduction to Git & GitHub

---

## 1.1 Overview

In any project—whether it involves writing software, creating documents, or designing content—files constantly change. New versions are created, edits are made, and old versions need to be preserved. Without an organized system, it becomes difficult to keep track of which version is the latest and who made specific changes.

**Git** and **GitHub** solve this problem by providing a structured way to manage versions and collaborate efficiently.

---

## 1.2 What is Git?

**Git** is a **version control system (VCS)**. A version control system records changes to files over time. It allows users to:

- Keep a complete history of every version of their files.
- Restore any previous version at any time.
- Work on different versions (branches) simultaneously without interfering with the main work.

Git stores these changes in a **repository**. A repository is like a project folder that contains all the files and a full record of every change made to them.

### Key Features of Git
- **Version Tracking** – Every saved change (commit) is recorded with a message and author information.  
- **Branches** – Users can create separate lines of work (branches) to experiment or develop features.  
- **Local Control** – Git works on your computer, even without an internet connection.

> **Example:**  
> A Business Analyst working on a Business Requirement Document (BRD) can use Git to save every update as a new version.  
> If an error is introduced, the analyst can return to an earlier version without searching through multiple “Final_v1” or “Final_v2” files.

---

## 1.3 What is GitHub?

While Git manages versions locally on your computer, projects often require collaboration with others. This is where **GitHub** plays an important role.

**GitHub** is a **cloud-based platform** that hosts Git repositories online. It allows teams to store their repositories in one central place where everyone can access and contribute.

### Key Features of GitHub
- **Centralized Storage** – All versions of the project are securely stored online.  
- **Collaboration** – Multiple users can work on the same project at the same time.  
- **Change Review** – Team members can review, comment, and suggest changes before they are merged.  
- **Integration** – GitHub connects with many tools used by developers, testers, and analysts.

> **Example:**  
> A QA engineer can create and update test scripts in a GitHub repository while developers simultaneously work on the code. Both can see and review each other’s changes without conflicts.

---

## 1.4 Who Uses Git & GitHub?

Git is widely adopted across different roles in technology projects:

- **Developers** – to manage source code and coordinate feature development.  
- **Quality Assurance (QA) Engineers** – to version control test scripts and automation frameworks.  
- **Business Analysts (BAs)** – to manage versions of BRDs, PRDs, and requirement documents.  
- **Designers & Content Creators** – to track revisions of designs and documentation.

These tools are not limited to technical users. Anyone managing documents that evolve over time can benefit from version control.

---

## 1.5 Why Use Git & GitHub?

The advantages of using Git and GitHub include:

1. **Elimination of File Chaos** – No more confusing file names like `document_final_v2.docx`.  
2. **Safe Collaboration** – Multiple people can work on the same project without overwriting each other’s work.  
3. **Clear Change History** – You can always identify what changed, who made the change, and when it happened.  
4. **Safe Experimentation** – Branches allow users to test new ideas without affecting the main version.  
5. **Industry Standard** – Git is used in nearly all modern projects, making it an essential skill.

---

## 1.6 Summary

- **Git** is a version control system that tracks file changes locally.  
- **GitHub** is an online service that stores Git repositories and enables team collaboration.  
- Together, they provide a reliable, efficient, and professional way to manage projects, whether they involve code, documents, or designs.
