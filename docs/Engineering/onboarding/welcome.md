# 👋 Welcome to Prosperna Engineering!

Welcome to the Prosperna Engineering Team! We're excited to have you onboard and can't wait to see what you'll build with us.

This document will give you a high-level overview of how we work, what we value, and how you can quickly get productive within your first few weeks.

---

## 🚀 Our Mission

At Prosperna, our goal is to empower SMEs with simple, powerful, and integrated tools that help them thrive in the digital economy. As an engineering team, we play a critical role in building and scaling these solutions.

---

## 🧭 What This Team Believes

We believe in:

- **Speed with quality**: Deliver fast but don’t compromise code integrity.
- **Automation-first mindset**: Every repeatable task is an opportunity to automate.
- **DevSecOps culture**: Security and reliability are everyone’s responsibility.
- **Knowledge sharing**: If it's not documented, it doesn't exist.
- **Continuous learning and feedback**: We get better every sprint, every review.

## 🧑‍🤝‍🧑 Our Team Structure

We operate with the following core squads, each with a distinct mission:

### 🏗 Platform Engineering
Focus: CI/CD automation, Infrastructure as Code (IaC), internal tooling, and security pipelines.

Responsibilities:
- Build and maintain GitHub Actions workflows
- Manage Terraform, Helm, Kubernetes, and provisioning tools
- Enforce DevSecOps practices across the stack
- Own internal developer platform and deployment reliability

### 🛠 Product Engineering
Focus: Building features that customers directly interact with — across frontend and backend microservices.

Responsibilities:
- Design and build backend services and APIs
- Implement frontend experiences using React/Next.js or Vue
- Own authentication, billing, order management, etc.
- Participate in architecture and tech decisions

### ✅ QA & Test Automation
Focus: Ensuring code quality and product stability before deployment.

Responsibilities:
- Create automated test suites (regression, functional, integration, e2e)
- Own regression testing in staging
- Monitor test coverage in CI
- Maintain test frameworks like Jest, Cypress, or Playwright

### 📊 Data & Analytics
Focus: Making our platform data-driven by building analytics, reporting, and decision support systems.

Responsibilities:
- Build ETL pipelines and data models
- Create dashboards (e.g., Metabase, Redash)
- Monitor key business metrics
- Ensure data quality and integrity

### 🧰 Support Engineering
Focus: Debugging, hotfixes, technical customer support, and production monitoring.

Responsibilities:
- Triage urgent issues and bugs reported from customer success
- Maintain uptime and incident response (SRE-lite)
- Work with Product Engineering for escalated cases
- Build internal tools to reduce support load

:::tip[Tip]

> 📌 All squads operate using a **Kanban flow** in **ClickUp**, which we use to manage deliverables, documentation, and internal discussions.

:::
