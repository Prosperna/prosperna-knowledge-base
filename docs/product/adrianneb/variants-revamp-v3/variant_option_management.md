#### **UC 14 | Add a New Variant Option**

| **Field**           | **Description**                                                                                                                                                     |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 14                                                                                                                                                               |
| Prepared By         | BA                                                                                                                                                                  |
| Last Updated        | September 16, 2025                                                                                                                                                  |
| Objectives          | The merchant should be able to add a new option to a variant type.                                                                                                  |
| Actor               | Merchant, System                                                                                                                                                    |
| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                                     |
|                     | A variant type card exists.                                                                                                                                         |
| Steps               | 1\. The system displays an empty, placeholder option field at the bottom of the "Options" list for a variant type.                                                  |
|                     | 2\. The merchant clicks into the placeholder field and begins typing a name (e.g., "Small").                                                                        |
|                     | 3\. The merchant either presses Enter or clicks outside the field (onBlur).                                                                                         |
|                     | 4\. If the field has a value, the system converts the placeholder into a standard, populated option field.                                                          |
|                     | 5\. The system immediately adds a new, empty placeholder option field below the newly created option.                                                               |
|                     | 6\. The system automatically creates new variant combinations in real-time (via incremental expansion) to include new combinations based on the newly added option. |
| Postconditions      | A new, named option is added to the variant type.                                                                                                                   |
|                     | A new placeholder option is available for input.                                                                                                                    |
|                     | The combinations table is updated.                                                                                                                                  |
| Business Trigger    | The merchant needs to expand the choices for a specific dimension of variation (e.g., adding a new size "XL").                                                      |
| Acceptance Criteria | Pressing Enter or blurring the field (with a value) converts the placeholder into a populated option and creates a new placeholder below.                           |
|                     | The Variant Combinations table correctly reflects the new combinations.                                                                                             |
| Estimates           |                                                                                                                                                                     |
| Success Message     |                                                                                                                                                                     |
| Error Messages      | (See UC 18 for option limit errors)                                                                                                                                 |

| **Business Rules/Desired Behavior**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| - The placeholder field is converted to a populated option field only when:<br />1. The field has a value (not empty), AND<br />2. Either the merchant presses Enter key OR the field loses focus (onBlur event)<br />- If the merchant types a value but neither presses Enter nor moves focus away, the field remains a placeholder and no new placeholder appears below.<br />- If the merchant leaves a field empty and moves focus away (onBlur with no value), the field remains a placeholder and is not converted to a populated option. |

#### **UC 18 | Block Adding Variant Option when Combination Limit is Almost Reached**

| **Field**           | **Description**                                                                                                                                                                                                                                                                                                                                                                   |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 18                                                                                                                                                                                                                                                                                                                                                                             |
| Prepared By         | BA                                                                                                                                                                                                                                                                                                                                                                                |
| Last Updated        | October 26, 2025                                                                                                                                                                                                                                                                                                                                                                  |
| Objectives          | The system should strictly enforce the combination limit according to the merchant's subscription plan by controlling whether the empty placeholder option field appears. A placeholder should only appear if adding another option to that specific variant type would keep the total possible combinations within the plan limit (Free: 50, Plus: 100, Pro: 120, Premium: 150). |
| Actor               | System, Merchant                                                                                                                                                                                                                                                                                                                                                                  |
| Preconditions       | The merchant is on the Add/Edit Product page.                                                                                                                                                                                                                                                                                                                                     |
|                     | At least one variant type exists with one or more options.                                                                                                                                                                                                                                                                                                                        |
|                     | The system has calculated the current total number of variant combinations.                                                                                                                                                                                                                                                                                                       |
|                     | The merchant's plan determines the maximum combination limit (Free: 50, Plus: 100, Pro: 120, Premium: 150).                                                                                                                                                                                                                                                                       |
| Steps               | 1\. The system continuously calculates the potential number of combinations if an additional option were added to each variant type.<br />Formula:<br />potential = (total_combinations / n_k) × (n_k + 1)<br />Where:<br />- n_k = number of options in variant type k<br />- plan_limit = merchant's plan limit (Free: 50, Plus: 100, Pro: 120, Premium: 150)                   |
|                     | 2\. For each variant type:<br />**If potential ≤ plan_limit:** The system displays the placeholder option field for that variant type.<br />**If potential > plan_limit:** The system hides the placeholder option field for that variant type.                                                                                                                                   |
|                     | 3\. The merchant can only add a new option to types where the placeholder is visible.                                                                                                                                                                                                                                                                                             |
| Postconditions      | The system never allows the total variant combinations to exceed the merchant's plan limit (Free: 50, Plus: 100, Pro: 120, Premium: 150).                                                                                                                                                                                                                                         |
|                     | Placeholders only appear where valid.                                                                                                                                                                                                                                                                                                                                             |
| Business Trigger    | The system must proactively enforce the plan-specific combination cap by preventing merchants from adding options that would exceed their plan's limit.                                                                                                                                                                                                                           |
| Acceptance Criteria | A placeholder option field is only displayed if adding an option to that variant type keeps total combinations within the plan limit (Free: 50, Plus: 100, Pro: 120, Premium: 150).                                                                                                                                                                                               |
|                     | If adding an option would exceed the plan's combination limit, no placeholder is shown for that type.                                                                                                                                                                                                                                                                             |
|                     | Reducing options (and thus total combinations) restores placeholders where valid.                                                                                                                                                                                                                                                                                                 |
| Estimates           |                                                                                                                                                                                                                                                                                                                                                                                   |
| Success Message     |                                                                                                                                                                                                                                                                                                                                                                                   |
| Error Messages      |                                                                                                                                                                                                                                                                                                                                                                                   |

| **Business Rules/Desired Behavior**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Important Note:** The calculation ensures combinations stay exactly at or below the plan limit.<br />**Calculation Logic:**<br />`plan_limit = get_plan_limit(merchant_plan)`<br />`total_combinations = 1`<br />`for each variant_type in variant_types:`<br />`total_combinations *= variant_type.option_count`<br />`for each variant_type in variant_types:`<br />`n_k = variant_type.option_count`<br />`potential = (total_combinations / n_k) * (n_k + 1)`<br />`if potential <= plan_limit:`<br />`show_placeholder(variant_type)`<br />`else:`<br />`hide_placeholder(variant_type)`<br />The system calculates `total_combinations` by multiplying the option counts of all variant types. For each type, it calculates `potential` combinations if one more option is added. Placeholder visibility depends on whether `potential` remains within the plan limit.<br />**Examples:**<br />**Scenario: Pro Plan (120 combination limit)**<br />Current setup:<br />- Size: 5 options<br />- Color: 4 options<br />- Material: 2 options<br />- Finish: 3 options<br />→ Total = 5 × 4 × 2 × 3 = **120 combinations**<br />Adding one option:<br />- To Size: (120 / 5) × 6 = 144 (invalid - exceeds 120)<br />- To Color: (120 / 4) × 5 = 150 (invalid - exceeds 120)<br />- To Material: (120 / 2) × 3 = 180 (invalid - exceeds 120)<br />- To Finish: (120 / 3) × 4 = 160 (invalid - exceeds 120)<br />**Scenario: Premium Plan (150 combination limit)**<br />Same setup as above (120 combinations):<br />Adding one option:<br />- To Size: (120 / 5) × 6 = 144 (valid)<br />- To Color: (120 / 4) × 5 = 150 (valid)<br />- To Material: (120 / 2) × 3 = 180 (invalid)<br />- To Finish: (120 / 3) × 4 = 160 (invalid) |

#### **UC 19 | Block Saving with Null Variant Option Name**

| **Field**           | **Description**                                                                                                                                                                                                                                                                                                        |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 19                                                                                                                                                                                                                                                                                                                  |
| Prepared By         | BA                                                                                                                                                                                                                                                                                                                     |
| Last Updated        | September 21, 2025                                                                                                                                                                                                                                                                                                     |
| Objectives          | The system should prevent saving the variant configuration if a variant option's name is empty. The system should not auto-exclude the option or discard its associated combinations, but instead call out the empty field with inline validation until the merchant either corrects or deletes the option explicitly. |
| Actor               | System, Merchant                                                                                                                                                                                                                                                                                                       |
| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                                                                                                                                                                                        |
|                     | At least one "Variant Option" field exists that has been converted from a placeholder to a populated option (via onBlur + hasValue or onEnter + hasValue) but was subsequently cleared, leaving it with an empty name.                                                                                                 |
| Steps               | 1\. As the merchant types or clears the option field, the system provides real-time inline validation:<br /> - Outlines the empty input field in red.<br /> - Displays red error text "Required\*" below the field.                                                                                                    |
|                     | 2\. The merchant clicks the "Save" button.                                                                                                                                                                                                                                                                             |
|                     | 3\. The system detects the empty option name.                                                                                                                                                                                                                                                                          |
|                     | 4\. The system will then:<br /> - Block the save operation.<br /> - Ensure the inline validation error is visible.<br /> - Display a general error toast: "Please complete all the required fields."                                                                                                                   |
| Postconditions      | The variant configuration is not saved.                                                                                                                                                                                                                                                                                |
|                     | Existing variant combinations and their data are preserved until the merchant explicitly deletes the option.                                                                                                                                                                                                           |
| Business Trigger    | The system must enforce data integrity, as an unnamed option is meaningless, but it should not destroy combinations unless deletion is explicit.                                                                                                                                                                       |
| Acceptance Criteria | The save action fails.                                                                                                                                                                                                                                                                                                 |
|                     | The specific input field is outlined in red with the "Required\*" message.                                                                                                                                                                                                                                             |
|                     | The error toast "Please complete all the required fields." is displayed.                                                                                                                                                                                                                                               |
| Estimates           |                                                                                                                                                                                                                                                                                                                        |
| Success Message     |                                                                                                                                                                                                                                                                                                                        |
| Error Messages      | "Please complete all the required fields."                                                                                                                                                                                                                                                                             |

| **Business Rules/Desired Behavior**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - A variant option field is considered "populated" (and thus subject to this validation) if it was created through the onBlur + hasValue or onEnter + hasValue trigger from UC 14.<br />- Placeholder fields (those that have not yet been converted to populated options) are not subject to this validation and can remain empty without blocking save.<br />- Once a field has been converted from placeholder to populated option, it cannot revert back to being a placeholder. If cleared, it becomes a populated option with an empty name, which triggers this validation. |
