#### **UC 01 | Enable Variants**

| **Field**           | **Description**                                                                                                                                                                                                                                                                                                                                                                                                       |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 01                                                                                                                                                                                                                                                                                                                                                                                                                 |
| Prepared By         | BA                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Last Updated        | September 16, 2025                                                                                                                                                                                                                                                                                                                                                                                                    |
| Objectives          | The merchant should be able to enable the variants feature for a product for the first time.                                                                                                                                                                                                                                                                                                                          |
| Actor               | Merchant, System                                                                                                                                                                                                                                                                                                                                                                                                      |
| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                                                                                                                                                                                                                                                                                       |
|                     | The "Product Variants" toggle button is turned off.                                                                                                                                                                                                                                                                                                                                                                   |
|                     | There are no existing variant types for the product.                                                                                                                                                                                                                                                                                                                                                                  |
| Steps               | 1\. The system displays the "Product Variants" section with an empty-state screen containing:<br />    - Header: "No Variants Created"<br />    - Subtext: "A variant is a specific version of a product with unique attributes like size, color, or material - for example, a t-shirt in 'Large/Red' or 'Medium/Blue' would be separate variants of the same base product."<br />    - CTA Button: "Enable Variants" |
|                     | 2\. The merchant clicks the "Enable Variants" button (or toggle button).                                                                                                                                                                                                                                                                                                                                              |
|                     | 3\. The system will then:<br />    - Turn on the "Product Variants" toggle button.<br />    - Automatically create and display one empty "Variant Type Card".<br />    - Display the "Variant Combinations" section with its own empty state ("No Variants Yet").                                                                                                                                                     |
| Postconditions      | The Variants feature for the product is successfully enabled.                                                                                                                                                                                                                                                                                                                                                         |
|                     | One empty variant type card is displayed, ready for configuration.                                                                                                                                                                                                                                                                                                                                                    |
| Business Trigger    | The merchant wants to add variations to their product for the first time.                                                                                                                                                                                                                                                                                                                                             |
| Acceptance Criteria | The "Product Variants" toggle button is successfully turned on.                                                                                                                                                                                                                                                                                                                                                       |
|                     | An empty "Variant Type Card" is displayed.                                                                                                                                                                                                                                                                                                                                                                            |
|                     | The "Variant Combinations" section is displayed.                                                                                                                                                                                                                                                                                                                                                                      |
| Estimates           |                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Success Message     |                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Error Messages      |                                                                                                                                                                                                                                                                                                                                                                                                                       |

| **Business Rules/Desired Behavior**                                                                                                                                                                       |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - This flow describes the first-time activation. If a merchant disables and later re-enables variants, the system should restore the previously saved configuration instead of creating a new empty card. |

#### **UC 02 | Disable Variants**

| **Field**           | **Description**                                                                                                                                                                                                |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 02                                                                                                                                                                                                          |
| Prepared By         | BA                                                                                                                                                                                                             |
| Last Updated        | September 16, 2025                                                                                                                                                                                             |
| Objectives          | The merchant should be able to disable the variants feature of a product.                                                                                                                                      |
| Actor               | Merchant, System                                                                                                                                                                                               |
| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                                                                                |
|                     | The "Product Variants" toggle button is turned on.                                                                                                                                                             |
| Steps               | 1\. The merchant navigates to the "Product Variants" section.                                                                                                                                                  |
|                     | 2\. The merchant clicks the "Product Variants" toggle button to turn it off.                                                                                                                                   |
|                     | 3\. The system will then:<br />    - Turn off the toggle button.<br />    - Hide the "Variant Configuration" section (where variant type cards are managed).<br />    - Hide the "Variant Combinations" table. |
| Postconditions      | The merchant has successfully disabled the variants feature of a product.                                                                                                                                      |
|                     | The product will behave as a single product in the storefront.                                                                                                                                                 |
| Business Trigger    | The merchant wants to temporarily or permanently remove the variations of their product from the storefront.                                                                                                   |
| Acceptance Criteria | The "Product Variants" toggle button is successfully turned off.                                                                                                                                               |
|                     | The "Variant Configuration" and "Variant Combinations" sections are hidden from view.                                                                                                                          |
| Estimates           |                                                                                                                                                                                                                |
| Success Message     |                                                                                                                                                                                                                |
| Error Messages      |                                                                                                                                                                                                                |

| **Business Rules/Desired Behavior**                                                                                                                                                                                 |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - Disabling variants does not delete the variant configuration data. If the merchant re-enables the variants, the previously saved types, options, and combination data should be restored.                         |

#### **UC 03 | Block Saving Product with Variants Enabled But No Combinations**

| **Field**           | **Description**                                                                                                                                      |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 03                                                                                                                                                |
| Prepared By         | BA                                                                                                                                                   |
| Last Updated        | September 16, 2025                                                                                                                                   |
| Objectives          | The system should block any saving attempts on a product when the variants feature is enabled but no valid variant combinations have been generated. |
| Actor               | System, Merchant                                                                                                                                     |
| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                      |
|                     | The "Product Variants" toggle button is turned on.                                                                                                   |
| Steps               | 1\. The merchant has not created any variant types.                                                                                                  |
|                     | 2\. The merchant clicks the main "Save" button for the product.                                                                                      |
|                     | 3\. The system detects that the variants toggle button is on, but no variant types have been created.                                                |
|                     | 4\. The system will then:<br />    - Not save the product.<br />    - Display an error toast: "Variant type and option is required."                 |
| Postconditions      | The system has successfully blocked the product from being saved with an incomplete variant configuration.                                           |
| Business Trigger    | The system needs to enforce data integrity to prevent products from being sold with incomplete variant information.                                  |
| Acceptance Criteria | The product is not saved.                                                                                                                            |
|                     | The correct error toast message is displayed.                                                                                                        |
| Estimates           |                                                                                                                                                      |
| Success Message     |                                                                                                                                                      |
| Error Messages      | "Variant type and option is required."                                                                                                               |

| **Business Rules/Desired Behavior**                                                                                          |
| ---------------------------------------------------------------------------------------------------------------------------- |
| - This validation ensures that any product sold with variants has at least one selectable variation.                         |
