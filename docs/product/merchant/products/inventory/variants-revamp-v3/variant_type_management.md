#### **UC 04 | Add a New Variant Type**

| **Field**           | **Description**                                                                                                                                                                                                                                                                                |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 04                                                                                                                                                                                                                                                                                          |
| Prepared By         | BA                                                                                                                                                                                                                                                                                             |
| Last Updated        | September 16, 2025                                                                                                                                                                                                                                                                             |
| Objectives          | The merchant should be able to add a new variant type card to the configuration.                                                                                                                                                                                                               |
| Actor               | Merchant, System                                                                                                                                                                                                                                                                               |
| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                                                                                                                                                                |
|                     | The "Product Variants" feature is enabled.                                                                                                                                                                                                                                                     |
|                     | All existing variant type cards (if any) have both a variant type name and at least one option entered.                                                                                                                                                                                        |
| Steps               | 1\. The merchant navigates to the "Variant Configuration" section.                                                                                                                                                                                                                             |
|                     | 2\. The merchant clicks the "Add Variant Type" button.                                                                                                                                                                                                                                         |
|                     | 3\. The system verifies that all existing variant type cards are complete (have a type name and at least one option).                                                                                                                                                                          |
|                     | 4\. The system checks if the current number of variant types is less than the maximum limit (Free: 2, Plus: 3, Pro: 4, Premium: 6).                                                                                                                                                            |
|                     | 5\. The system adds a new, empty "Variant Type Card" to the bottom of the list. This card includes:<br /> - An empty "Variant Type Name" input field.<br />\- Display Type section with "Text" card selected by default<br /> - An "Options" section with one empty, placeholder option field. |
| Postconditions      | A new variant type card is successfully added to the UI, ready for configuration.                                                                                                                                                                                                              |
| Business Trigger    | The merchant needs to define a new dimension of variation for their product (e.g., adding "Material" after already defining "Color" and "Size").                                                                                                                                               |
| Acceptance Criteria | A new, empty "Variant Type Card" is visible on the screen.                                                                                                                                                                                                                                     |
| Estimates           |                                                                                                                                                                                                                                                                                                |
| Success Message     |                                                                                                                                                                                                                                                                                                |
| Error Messages      | (See UC 08, UC 09 for limit errors)                                                                                                                                                                                                                                                            |

| **Business Rules/Desired Behavior**                                                                                                                                                                                                                                                                                                                                                             |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - The "Add Variant Type" button should be disabled when any existing variant type card is incomplete (missing type name or has no options).<br />- The "Add Variant Type" button should be disabled if the variant type limit has been reached.<br />- This progressive completion approach ensures data integrity and prevents merchants from creating too many incomplete variant type cards. |

#### **UC 05 | Add a Color Swatch-Based Variant Type**

| **Field**           | **Description**                                                                                                                                                                                                                                          |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Use Case ID         | UC 05                                                                                                                                                                                                                                                    |
| Prepared By         | BA                                                                                                                                                                                                                                                       |
| Last Updated        | September 19, 2025                                                                                                                                                                                                                                       |
| Objectives          | The merchant should be able to designate a variant type for "Color Swatch", enabling a color picker for its options.                                                                                                                                     |
| Actor               | Merchant, System                                                                                                                                                                                                                                         |
| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                                                                                                                          |
|                     | A variant type card exists.                                                                                                                                                                                                                              |
|                     | No other variant type is currently designated as the "Color Swatch" type.                                                                                                                                                                                |
| Steps               | 1\. The merchant locates the variant type card they wish to designate for color.                                                                                                                                                                         |
|                     | 2\. The system displays a "Display Type" section with three selectable cards: "Text" (default), "Icon", and "Color Swatch".                                                                                                                              |
|                     | 3\. The merchant selects the "Color Swatch" card.                                                                                                                                                                                                        |
|                     | 4\. The system will then:<br /> - Update the UI for all options within that specific variant type card to include a color picker input field next to the option name.<br /> - Disable the "Color Swatch" card for all other existing variant type cards. |
| Postconditions      | The selected variant type is now a "Color Swatch" type.                                                                                                                                                                                                  |
|                     | All options for this variant type now have a color picker.                                                                                                                                                                                               |
| Business Trigger    | The merchant wants to provide visual color swatches for a specific variant type (e.g., "Main Color").                                                                                                                                                    |
| Acceptance Criteria | A color picker input appears next to each option in the designated variant type.                                                                                                                                                                         |
|                     | The "Color Swatch" card is disabled for all other variant types.                                                                                                                                                                                         |
| Estimates           |                                                                                                                                                                                                                                                          |
| Success Message     |                                                                                                                                                                                                                                                          |
| Error Messages      |                                                                                                                                                                                                                                                          |

| **Business Rules/Desired Behavior**                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - Only one variant type can be set to "Color Swatch" at a time to ensure a clean and unambiguous data structure. <br />- If the user switches the display type to "Text" or "Icon", the color data is preserved in the background and will reappear if they switch back to "Color Swatch".<br />- The default color of the color picker should be "#000000" (black)                         |

#### **UC 08 | Block Adding Variant Type when Type Limit is Reached**

|                     | 3\. The system disables the "Add Variant Type" button, making it unclickable and visually indicating it is disabled.                                                                                                                           |
|                     | **Condition 2: Merchant is on a Plus Plan**                                                                                                                                                                                                    |
|                     | 1\. The merchant has 3 existing variant types.                                                                                                                                                                                                 |
|                     | 2\. The system detects that the variant type limit for the Plus plan (3 types) has been reached.                                                                                                                                               |
|                     | 3\. The system disables the "Add Variant Type" button, making it unclickable and visually indicating it is disabled.                                                                                                                           |
|                     | **Condition 3: Merchant is on a Pro Plan**                                                                                                                                                                                                     |
|                     | 1\. The merchant has 4 existing variant types.                                                                                                                                                                                                 |
|                     | 2\. The system detects that the variant type limit for the Pro plan (4 types) has been reached.                                                                                                                                                |
|                     | 3\. The system disables the "Add Variant Type" button, making it unclickable and visually indicating it is disabled.                                                                                                                           |
|                     | **Condition 4: Merchant is on a Premium Plan**                                                                                                                                                                                                 |
|                     | 1\. The merchant has 6 existing variant types.                                                                                                                                                                                                 |
|                     | 2\. The system detects that the variant type limit for the Premium plan (6 types) has been reached.                                                                                                                                            |
|                     | 3\. The system disables the "Add Variant Type" button, making it unclickable and visually indicating it is disabled.                                                                                                                           |

| Business Trigger    | The system needs to enforce subscription plan limits.                                                                                                                                                                                          |

|                     | The button displays a disabled state (grayed out, non-clickable).                                                                                                                                                                              |
|                     | The button becomes enabled again if the merchant deletes a variant type and falls below the limit.                                                                                                                                             |
| Estimates           |                                                                                                                                                                                                                                                |
| Success Message     |                                                                                                                                                                                                                                                |
| Error Messages      |                                                                                                                                                                                                                                                |

| Preconditions       | The merchant is on the "Add/Edit Product" page.                                                                                                                                                                                                  |
|                     | At least one "Variant Type Card" has an empty "Variant Type Name" input field.                                                                                                                                                                   |

|                     | 3\. The system detects that a "Variant Type Name" field is null/empty.                                                                                                                                                                           |
|                     | 4\. The system will then:<br /> - Block the save operation.<br /> - Ensure the inline validation error is visible.<br /> - Display a general error toast: "Please complete all the required fields."                                             |
| Postconditions      | The variant configuration is not saved.                                                                                                                                                                                                          |
|                     | Existing variant combinations and their data are preserved until the merchant explicitly deletes the variant type.                                                                                                                               |

|                     | The specific input field is outlined in red with the "Required\*" message.                                                                                                                                                                       |
|                     | The error toast "Please complete all the required fields." is displayed.                                                                                                                                                                         |
| Estimates           |                                                                                                                                                                                                                                                  |
| Success Message     |                                                                                                                                                                                                                                                  |
| Error Messages      | "Please complete all the required fields."                                                                                                                                                                                                       |

| **Business Rules/Desired Behavior**                                                                                                                                                                                                                                                                    |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| - The system should not auto-exclude a variant type or its combinations when the name is emptied. Combinations and their data remain intact until the merchant deletes it.<br />-This validation should be triggered upon attempting to save the entire variant configuration.                         |
